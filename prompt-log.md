## 第一步：项目初始化与基础结构搭建
第一步：喵记 - 带虚拟猫咪互动的治愈系动画日记网站 - 项目初始化与基础结构搭建
请严格按照以下要求生成完整可运行的纯前端项目，仅使用 HTML5、CSS3、原生 JavaScript，不引入任何框架、不搭建后端、不使用云端存储：
项目文件结构
主入口文件：index.html（单页应用）
全局样式文件：style.css
逻辑脚本文件：script.js
静态资源目录：assets/anim（专门存放猫咪 Lottie 动画、图标、背景图）
开发日志目录：logs/prompt-log.md（用于记录所有 AI 提示词、修改记录和调试思路）

HTML 语义化结构
使用标准 HTML5 文档声明
配置完整的 meta 标签，包括 viewport 视口标签确保移动端适配
页面标题设置为：喵记 - 治愈系动画日记
必须包含以下 5 个核心容器，每个容器都要有唯一 ID 和清晰的 class 命名：
#app：页面整体根容器
.editor-wrap：日记编辑区域容器
.diary-list-wrap：历史日记列表容器
#cat-animation：虚拟猫咪动画专属容器

所有标签语义化，添加必要的注释说明
CSS 全局样式规范

定义治愈系低饱和度全局 CSS 变量：
--bg: #f8f5f2（页面背景色）

--main: #e8a87c（主色调）
--light: #f3d4b7（浅色调）
--radius: 12px（统一圆角）
--shadow: 0 2px 10px rgba (0,0,0,0.05)（统一阴影）
整体页面居中布局，设置最大宽度限制

#bubble 容器固定定位在页面右上角，层级 z-index: 15，不遮挡输入框
统一所有元素的间距、圆角、阴影风格，保持整体治愈系视觉效果
搭建响应式布局基础框架
JavaScript 基础模块封装
封装完整的 LocalStorage 工具类，包含以下方法：
saveData (key, data)：保存数据到本地存储
loadData (key, defaultValue)：从本地存储读取数据，支持默认值兜底
removeData (key)：删除指定键的数据
所有数据操作都添加 try-catch 异常捕获和 JSON 格式校验
页面加载时自动初始化本地存储数据结构
预留输入、保存、删除、清空四个核心事件的监听入口

最终交付要求
生成的代码可以直接在浏览器中打开运行
结构清晰，便于后续功能迭代开发

### 第二步-3：实现保存按钮与LocalStorage存储
实现日记保存功能，严格使用第一步封装的 LocalStorage 工具类：
 1. 点击 "保存日记" 按钮，收集标题、正文和当前时间 
2. 数据结构：{id: 时间戳，title, content, createTime:"YYYY-MM-DD HH:mm"} 
3. 保存到 LocalStorage 的 "diaryList" 数组中

## 第五步：猫咪互动动画基础逻辑
### 第五步-1：实现猫咪待机动画
     实现猫咪基础待机动画： 
1. 使用 CSS @keyframes 实现猫咪轻微上下浮动的待机动画
 2. 动画时长 3 秒，无限循环，缓动效果 
3. 页面加载完成后自动开始播放待机动画 
4. 动画效果柔和，不刺眼
 5. 确保不影响页面其他功能       
 
### 第五步-2：绑定保存事件的动画切换
     绑定保存事件与猫咪动画：
 1. 当用户点击 "保存日记" 按钮时
 2. 猫咪停止待机动画，切换为开心的动画状态 
3. 开心状态持续 2 秒后，自动恢复为待机动画
 4. 同时给出保存成功的视觉反馈 
5. 确保多次点击不会导致动画冲突   
     
### 第五步-3：绑定删除与输入事件的动画
     绑定更多事件与猫咪动画：
 1. 当用户点击 "删除日记" 按钮时，猫咪切换为难过的动画状态 
2. 当用户在输入框中输入文字时，猫咪切换为认真倾听的动画状态 
3. 每个动画状态持续 2 秒后自动恢复待机 
4. 添加动画队列，避免多个动画同时播放
 5. 所有动画使用原生 CSS 实现，无第三方库
